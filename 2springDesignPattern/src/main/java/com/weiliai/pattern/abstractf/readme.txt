抽象工厂模式(Abstract Factory Pattern)是指提供一个创建一系列相关或相互依赖对象的接口,无须指定他们的具体类.
客户端(应用层)不依赖于产品类实例如何被创建,如何被实现等细节,强调的是一系列相关的产品对象(属于同一产品族)一起使用创建对戏那个需要大量重复代码.

产品等级结构和产品族:
    1.产品等级结构--->工厂中各种各样的等级产品
    2.同一个工厂中各种各样的产品统称--->产品族
    3.产品工厂->具体的抽象工厂实现类

缺点:
    1.规定了所有可能被创建的产品集合,产品族中扩展产品困难,需要修改抽象工厂接口
    2.增加了系统的抽象性和理解难度
